1️⃣ 产品目标

- 目标：帮助用户完成长期目标
- 核心机制：计划生成 + 打卡 + 积分 + 奖励兑换

2️⃣ 用户流程

1. 用户输入目标（例如减肥）
2. 系统提问澄清（当下状态，目标）
3. 系统生成计划草案
4. 用户修改
5. 用户确认
6. 系统创建专属数据库
7. 每天展示今日任务
8. 用户打卡
9. 系统算分
10. 积分可兑换奖励

3️⃣ 积分规则
- 可由顾客设定，例如有些只有完成和未完成，顾客能够设定几分，不过我们也要生成草案先
- 完美完成：3 分
- 普通完成：2 分
- 勉强完成：1 分
- 未完成：0 分

4️⃣ 任务频率规则
- 任务必须支持：
  - daily
  - weekly_n（每周 n 次）(用户一周可运动 3 次,系统根据剩余天数与剩余次数决定今天是否推荐)
  - specific_days（例如 Mon Wed Fri）
  - monthly
  - 某个日期

5️⃣ 奖励系统规则
-用户自定义奖励
- 奖励包含：
  - 名称
  - 所需积分
- 积分足够即可兑换
- 兑换记录保留

6️⃣ RAG 用途定义
- 计划生成时检索知识库
- 用户问“为什么这样安排”时可引用
- 用户问替代方案时检索

7️⃣ Agent 用途定义
- Agent 负责：
  - 多步生成计划
  - 检查计划冲突
  - 用户修改后重新生成
  - 确认后写入数据库
